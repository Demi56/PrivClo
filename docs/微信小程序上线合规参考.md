## 一、技术侧（上线与性能）

### 1. 代码与包体

- 已启用 **`lazyCodeLoading: "requiredComponents"`**（`app.json`），勿随意移除。
- 主包、每个分包 **≤ 2MB**；大图走 CDN/云存储，构建产物勿进 Git。
- 插件包、扩展库暂不支持按需注入；大能力优先放分包 + 分包异步化。

### 2. 权限与隐私接口

当前已声明（`app.json` → `permission`）：

| 权限 | 用途文案 |
|------|----------|
| `scope.userLocation` | 当地天气与穿搭推荐 |
| `scope.camera` | 扫描/拍摄录入衣物 |
| `scope.album` | 选择衣物图片 |

**要求**：调用前按需授权；在公众平台配置《用户隐私保护指引》并勾选对应接口；文案与真实用途一致，不得超范围收集。

### 3. 云开发与安全

- 和风、混元等密钥放在 **云函数环境变量**，禁止写入前端或提交仓库。
- 云函数应对用户输入做基本校验与限流，避免被滥刷 AI/扣积分。

## 二、运营规范（审核高频项）

依据 [平台运营规范](https://developers.weixin.qq.com/miniprogram/product/) 摘要，与 PrivClo 相关部分：

### 1. 基本信息与类目

- 名称/简介/logo 与功能一致，不含误导、侵权、「国家级」「最免费」等违规用语。
- **服务类目**须覆盖：衣橱管理、穿搭记录、AI 推荐、积分/任务（若上线）等实际能力；避免「类目不符」下架（规范 5.7）。

### 2. 功能与体验

- **核心能力在首页体现**（规范 3.2）；避免审核员进入后只见占位或「暂未开发」。
- 功能与简介一致，无隐藏类目（规范 3.1）。
- 需提供**侵权/违规投诉机制**（规范 3.10）——设置页入口、隐私条款中说明。

### 3. 分享、积分、邀请（⚠️ 后续重点）

若上线积分任务、邀请好友、分享得奖励：

- 禁止 **强制分享** 才能继续、暗示分享（红点/动态按钮）、分享朋友圈获利（规范 5.1）。
- 「利益」含 **积分、优惠券、虚拟奖品**；规则须清晰、显著，禁止概率性/非即时获利/组队超 5 人等。
- 禁止 **多级分销、代理组队返利**（规范 5.9）。

**决策提醒**：设计「邀请有礼」「分享得积分」前，必须先过运营规范 5.1/5.9，否则可能封分享或支付能力。

### 4. 虚拟支付与积分购买（⚠️ 后续重点）

规范 **5.13**：小程序内虚拟商品（含 **虚拟代币、积分、解锁功能、订阅、付费功能** 等）须使用 **小程序支付**，不得引导至 App、H5、公众号、个人号等外部支付。

**决策提醒**：

- 用户 **人民币购买积分** → 需商户号 + `wx.requestPayment`（或云开发 `cloudPay`），并在类目与资质上合规。
- 仅用积分兑换功能、无现金充值 → 仍建议在用户协议中说明积分规则，避免被认定为变相付费未接入。
- iOS 对虚拟商品政策严格，提审前核对当期平台说明。

### 5. 用户内容与 AI（⚠️ 本项目相关）

- **日记/贴纸/照片** 属用户产生内容：需内容安全策略（敏感词、举报、违规处理说明）。
- 规范 **5.12.15**：不得通过上传照片、识别人像等 **恶意收集隐私**（如未告知的评分、算命类）。
- AI 抠图、穿搭建议：在《隐私政策》《AI 服务说明》中写清数据用途、存储期限、是否出境（若用第三方 API）。

### 6. 其他禁止行为

- 审核版与线上版不一致、按用户群体展示不同类目（绕审，规范 5.27 等）。
- 混淆腾讯/微信官方（登录页、logo、文案）。
- 支付完成页跳转无关公众号文章（规范 5.30）。
- 诱导跳转下载 App 并用积分/现金利诱（规范相关条款）。

## 三、PrivClo 当前状态与提审前清单

### 已较好对齐

- 分包结构清晰；`lazyCodeLoading` 已开。
- 权限说明与相机/相册/定位场景基本匹配。
- 已有隐私条款、AI 说明、游客模式保存拦截（`requireGuestLoginForSave`）。

- [ ] 公众平台类目、简介、截图与真实功能一致  
- [ ] 《用户隐私保护指引》已配置并含相机/相册/定位/用户信息（若用）  
- [ ] 隐私政策、用户协议可打开且与实现一致  
- [ ] 无未上线功能的误导性入口（或标注「演示」）  
- [ ] 云函数与第三方 AI 合规说明完整  
- [ ] 若上线积分购买：支付接入 + 活动规则页 + 无诱导分享  
- [ ] 主包/分包体积扫描通过  
- [ ] 真机：登录、支付（若有）、相机、定位、保存衣橱

## 四、后续开发时「须提醒用户」的决策类型

在 Cursor 协作中，出现以下需求时应 **主动提示审核/合规风险**：

1. 积分/会员/解锁功能的 **现金支付** 或跳转外部支付  
2. 分享得积分、邀请裂变、抽奖、拼团  
3. 收集手机号、身份证、通讯录、人脸生物识别  
4. 用户日记/图片 **公开展示、社交广场**（UGC 审核义务加重）  
5. 接入 **直播、医疗、金融、新闻** 等强监管类目能力  
6. 虚拟支付改价、自动续费、未明示的 AI 深度合成内容  
7. 包体暴涨、主包塞入大量静态资源  
8. 使用未授权品牌词、微信 logo、仿官方登录页
