## 待完成的优化（需手动处理）

### 单张图片/音频不超过 200K

微信要求：**图片和音频资源大小应不超过 200K**。当前主包及子包中仍有大量图片超过此限制。

**推荐做法：**

1. **使用 TinyPNG 批量压缩**（https://tinypng.com）
   - 上传 `images/` 目录下的 PNG/JPEG
   - 下载压缩后的图片替换原文件

2. **使用 ImageOptim（Mac）**
   - 下载：https://imageoptim.com
   - 拖入 `images/` 目录进行无损压缩

3. **降低分辨率**
   - 小程序设计稿基准 750rpx，图片宽度 750px 通常足够
   - 可使用 `sips -Z 750 图片路径`（Mac）批量缩放

### 主包进一步瘦身（若仍超 1.5M）

若主包仍超 1.5M，可考虑：

1. **图片上传至 CDN**：将 `images/` 中图片上传至云存储，代码中改用网络 URL
2. **分包异步化**：将 model、role 等页面的部分资源改为按需加载
3. **移除未使用资源**：检查并删除未引用的图片和代码
